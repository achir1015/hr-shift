## 三、各功能說明

### 🏢 公司 / 部門管理

- 支援多公司建立（免填統編）  
- 每間公司可建立多個部門  
- 刪除公司會連帶刪除所有部門

### 👤 員工管理

- 實名制：真實姓名、身分證字號  
- 支援月薪制（正職）與時薪制（兼職）  
- 工時型態：正常工時 / 變形工時 / 兼職  
- 帳號權限：一般員工 / 管理者 / 外部帳號  
- 住址作為「在家上班打卡」的比對基準  
- 密碼以 MD5 加密儲存

### ⏰ 班別設定

- 設定班別名稱（早班 / 晚班 / 大夜班）  
- 每個班別有顏色標記，在排班格顯示  
- **調班防呆規則**：只有相同 `total_hours`（總時數）的班別才能互換  
- 可標記為「休假班」，不計入工時

### 📅 排班管理

- 支援任意區間：雙週 / 四週 / 八週  
- 班表狀態：**草稿 → 已發布**  
- **草稿**：管理者可自由修改  
- **已發布**：全數鎖定，員工才看得到  
- 以顏色格狀顯示，一目了然

### 📍 打卡紀錄

- 支援 GPS / WIFI 兩種打卡方式  
- 可標記異常打卡  
- 管理者可手動新增、刪除、標記異常  
- 一般員工僅能查看自己的紀錄

### 📆 行事曆同步

- 記錄員工每天「有空 / 沒空」狀態  
- 來源：Google 行事曆 / iOS 行事曆 / 手動  
- 支援批次設定（一次設定一段日期）  
- 老闆端只看到狀態，看不到私人行程明細  
- 沒空的員工在智能排班時會被略過

### 📊 月統計報表

- 自動結算：依排班明細計算工時、加班、假日出勤  
- 薪資試算：月薪制 / 時薪制自動套用不同費率  
- 狀態：試算中 → 鎖定（確認結算）  
- 鎖定後不會被自動結算覆寫

## 五、⚠️ 特別注意事項

### 伺服器環境

| 項目 | 說明 |
| :---- | :---- |
| PHP 版本 | 7.0.x（不支援 nullable type、void、mixed） |
| 資料庫 | MariaDB 10，Port **3307**（非預設 3306） |
| 舊版 MySQL 5 | Port 3306，本系統不使用 |
| 網路存取 | 透過 ngrok tunnel（ipmos.ngrok.app） |
| 內網 | AP 隔離，需接網路線才能直連 192.168.6.66 |

### config.php 每次更新必填

define('DB\_HOST', '127.0.0.1:3307'); // 不可改成 localhost

define('DB\_PASS', '您的密碼');        // 每次重新上傳都要填

### ngrok 斷線處理

- ngrok 斷線時，外部網址 ipmos.ngrok.app 會顯示 ERR\_NGROK\_3200  
- 需到 NAS 實體重啟 ngrok 服務  
- 建議設定 ngrok 開機自動啟動

### 密碼安全

- 員工密碼使用 MD5 加密（非最高安全等級）  
- 建議正式上線前修改 admin 預設密碼（config.php ADMIN\_PASS）  
- 測試員工預設密碼為 `1234`，上線前請通知員工修改

### 班表發布不可逆

- 班表一旦按下「發布」即鎖定，**無法再修改**  
- 請確認排班內容無誤後再發布

### setup.php 安全性

- 安裝完成後請透過 FTP 刪除 `setup.php`  
- 保留此檔案會有資料庫被重建的風險

### 月統計計算方式

- 自動結算依據 Schedule\_Detail 的班別時數加總  
- 加班費計算：超出 (工作天數 × 8\) 小時的部分 × 1.33 倍  
- 假日出勤：週六日出勤天數 × 2 倍日薪
